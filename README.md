# 初期コミット
## 1. サービス概要
このサービスは、日々の運動量をカロリーから距離に変換し、日本を旅するように健康管理ができるアプリです。
運動すればするほどキャラが成長し、ご当地アイテムを集めながら旅が進むため、楽しみながら運動習慣を継続できます。
運動不足に悩む人が、ゲーム感覚で運動を続けられる仕組みを提供します。

## 2. このアイデアはどこから生まれたか
2-1. きっかけとなった体験・感情
仕事や日常が忙しく、運動する機会がなかなか取れず、ジムに通っても長続きしない時期が続いていた。
「運動しなきゃ」と思っていても、気づけば運動不足になっている自分に気づき、焦りやモヤモヤを感じていた。
ただ目標を立てるだけでは続かず、何か楽しさや達成感がないと継続できないという実感があった。

2-2. なぜそれが「気になった」のか
単に目標を立てるだけでは自分は続かないと、過去の経験から分かっていた。
自分はゲーム性や成長要素があるとモチベーションが上がり、楽しみながら継続できるタイプだと感じていたため、
「運動にもゲームのような仕組みがあれば続けられるのでは？」という思いが強くなった。
運動不足を解消したい気持ちと、楽しさがないと続かないという自分の性格が結びつき、このアイデアが生まれた。

## 3. 課題の整理
3-1. 表に見えている困りごと
運動を続けるモチベーションが保てない

記録アプリは味気なくて続かない

運動不足を自覚しているが、行動に移せない

3-2. 本当に解決したい課題

　運動が習慣化しない

　達成感や楽しさが感じられない

　運動の成果が実感しづらいから

→ 本当に解決したい課題：
「運動の成果を実感し、楽しみながら継続できる仕組みがない」

## 7. MVPにおけるAI提案機能の扱いについて

本アプリでは「AIによる運動アイデア提案」を構想していますが、
MVP段階では以下の理由から **AI連携は実装対象外** とします。

- 外部API（例：OpenAI API）との連携は実装コストが高い
- API利用料が発生するため、個人開発の初期段階では負担が大きい
- MVPの価値は「運動記録 → カロリー → 距離 → 旅の進行」を体験できる点にあるため、
  AI提案がなくてもコア体験は成立する

### 🔹 本リリースでの実装予定
正式リリースでは以下のような形でAI提案機能を追加予定です。

- ユーザーの運動履歴をもとにした運動メニューの提案
- 旅の進捗に応じたおすすめ運動（例：山エリアなら登山系の運動）
- 1日の運動量が少ない場合の軽い運動の提案

### 🔹 MVPで実装する範囲
MVPでは以下の機能に絞ります。

- 運動記録の登録
- カロリー計算
- カロリー → 距離変換
- 旅の進捗表示
- キャラクターの成長

AI提案機能は「将来的な拡張機能」として位置づけ、  
MVPでは実装しない方針とします。

## 8. 運動記録の仕様（カロリー計算）
### カロリー計算式（MVP）
消費カロリー = METs × 60kg × (運動時間 / 60) × 1.05

### 種目ごとの METs（例）
| 種目 | METs |
|------|------|
| ウォーキング | 3.5 |
| ランニング | 7.0 |
| サイクリング | 6.0 |
| 筋トレ | 5.0 |
| ヨガ | 2.5 |

### カロリー → 距離変換（例）
距離(km) = カロリー / 50

## 9. このアプリの懸念点とその対策
懸念点①
何が問題になりそうか：運動を記録しないと旅が進まず、離脱する可能性

対策：
　- 小さな達成でも旅が進むように調整
　- ご当地アイテムで収集欲を刺激

## 10. 今後の展開（ユーザー体験の変化を含む）

MVP では「運動記録 → カロリー → 距離 → 旅の進行 → キャラ成長」という
コア体験に絞っていますが、今後は以下のような拡張を予定しています。

### ① 世界一周モード
- 日本一周後に世界各地へ旅できるモードを追加
- **ユーザーの変化：運動が“世界を旅する体験”に拡張され、継続意欲がさらに高まる**

### ② キャラクターの着せ替え機能
- 運動量や旅の達成度に応じて衣装やアクセサリーを解放
- **ユーザーの変化：自分の努力が見た目に反映され、愛着とモチベーションが向上する**

### ③ ご当地アイテムの収集
- 各地域の名物アイテムをコレクションできる
- **ユーザーの変化：旅の達成感が増し、運動が“収集ゲーム”としても楽しめる**

### ④ 友達と旅レース機能
- 友達と距離を競ったり、同じルートを一緒に進める
- **ユーザーの変化：運動が“ひとりの習慣”から“仲間と楽しむ体験”へ変わる**

### ⑤ AIによる運動提案（本リリースで実装予定）
- 運動履歴や旅の進捗に応じて最適な運動を提案
- **ユーザーの変化：運動の選択に迷わず、継続しやすくなる**


## 11. 技術スタック
11-1. 使用予定の技術
本アプリは Docker 上で動作する Rails アプリケーションです。

### バックエンド
- Ruby：3.2.2  
- Rails：7.1.x  
- PostgreSQL：15  
- RSpec：3.x  

### フロントエンド
- Node.js：18.x  
- Yarn：1.x  
- Hotwire（Turbo / Stimulus）  
- Tailwind CSS（予定）  

### インフラ / 開発環境
- Docker / Docker Compose  
- GitHub（バージョン管理）

11-2. 技術選定の理由
Rails：開発速度が速く、経験と相性が良い

PostgreSQL：扱いやすく本番運用向き

AI提案：ユーザーの継続を助けるため

不安点：旅ルートや距離計算のロジックが複雑化する可能性
